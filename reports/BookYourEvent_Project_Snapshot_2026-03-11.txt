3) How Auth Is Handled
============================================================
Auth strategy in this codebase:
- JWT-based auth.
- Frontend stores JWT and user object in localStorage keys: token, user.
- Axios request interceptor reads localStorage token and sends:
  Authorization: Bearer <token>
- Backend protect middleware reads token only from Authorization header and verifies with JWT_SECRET.
- 401 response interceptor clears localStorage token/user and redirects to /login?expired=1.
- Cookie-based session/JWT auth is NOT implemented in middleware.

Google OAuth path:
- Backend Google callback issues JWT and redirects to frontend with token + user in query params.
- Frontend GoogleAuthSuccess page reads params and calls AuthContext login(), which persists to localStorage.
